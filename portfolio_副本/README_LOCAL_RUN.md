# PortfolioRadar 本地 MVP 运行说明

这个文件夹里有一个单文件副本：

```bash
python3 portfolio/portfolio_radar_local_mvp.py
```

运行后浏览器会自动打开：

```text
http://127.0.0.1:8765
```

你可以在页面里：

1. 添加自己持有的基金代码和基金名称。
2. 尝试自动抓取基金持仓。
3. 如果自动抓取失败，手动粘贴持仓。
4. 点击“计算今日信号”查看 MVP 版建议。
5. 点击“查看判断依据”查看技术、事件、情绪、历史四类依据。
6. 点击“历史回顾”查看每次计算留下的历史记录和相关性样本状态。
7. 在右侧输入 DeepSeek API Key，手动触发“AI 分析当前基金新闻”。

第三版交互已经加入：

- 所有主要按钮都有加载、成功、失败提示。
- 保存持仓成功后会弹窗确认。
- 自动抓取失败会提示改用手动上传。
- 页面调整为更宽的居中双栏布局，右侧用于 AI 新闻判断、历史相关性分析和已隐藏基金。
- 每张基金卡片可以“隐藏基金”，右侧“已隐藏基金”里可以恢复；这只是软隐藏，不会清除持仓、信号或历史记录。
- 事件依据会展示命中的新闻标题和来源链接；如果周末或接口没有新闻，会明确提示保持中性。
- AI 新闻判断只要求输入 DeepSeek API Key；模型、接口、提示词和参数已经内置。API Key 只随本次请求发送，不写入 SQLite、不写入文件。
- API Key 默认每次使用后会自动清空。也可以勾选“仅本页面暂存到今日 15:00”，用于当天 12:00 / 14:40 盘中 AI 看盘；这只存在浏览器当前页面内存里，不写入 SQLite、不写入文件，刷新页面后失效。
- AI 分析支持“直接给结论”和“展开分析逻辑”两种偏好；详细模式会展示新闻筛选、相关性判断、影响路径和风险说明。
- 新闻源优先尝试东方财富财经新闻，再补充 AKShare 新闻聚合；如果来源只提供标题或摘要，AI 会被明确告知“未读取付费全文”，并降低置信度。
- 新闻发布时间会参与时效权重：T 日和 T-1 日权重较高，超过 3 天的新闻只作为背景信息。
- AI 判断会被限制在基金持仓股票及其所属领域上，不能因为泛市场关键词就直接判为相关。
- 新增“AI 看盘”：基于前十大持仓的走势、量能和主力资金流做技术/情绪辅助分析。
- 新增“事后回访”：当 AI 做出高置信方向判断后，后续可用持仓加权涨跌做复盘，发现“判断利好但连续几天收益为负”等情况时标记为未命中。
- 新增“回填近一周数据”：根据当前基金持仓股，拉取近一周日线、成交量、主力资金流，生成启动历史样本，减少首次使用时大面积“样本不足”。
- 本地服务运行期间会在大陆时间 12:00 和 14:40 尝试自动刷新行情/资金流并更新信号。电脑、本地 Python 服务和页面需要保持打开；本地 MVP 不具备服务器离线推送能力。
- 首页右侧有独立“历史相关性分析”专区，展示样本数、命中率、平均后续收益、综合相关系数；这里的样本指多次信号计算后的时间序列验证样本，不是单只基金的十只持仓股票。样本不足时会明确提示，不会伪造收益。

手动持仓格式示例：

```text
600188,22%
601898,0.15
601225,0.12
```

数据只保存在本机：

```text
data/portfolio_radar_local.db
```

按 `Ctrl+C` 可以停止本地服务。
